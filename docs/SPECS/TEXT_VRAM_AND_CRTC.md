# テキスト VRAM / uPD3301 CRTC / uPD8257 DMAC 仕様

PC-8801 のテキスト画面・アトリビュート・グラフィック画面まわりの仕様。
出典は主に <http://mydocuments.g2.xrea.com/html/p8/vraminfo.html>
(以下「vraminfo」と略) と各 LSI データシート、リファレンスエミュレータ
(BubiC / QUASI88 / X88000M) の実装コード。

> **位置づけ**: 本ドキュメントは PC-8801 ハードウェア仕様の解説。
> Bubilator88 自体の実装状態は §11「実装サマリ」と
> `memory/project_vram_spec_audit.md` を参照。

---

## 1. テキスト VRAM 配置

### 1.1 アドレス

| モード | デフォルト先頭アドレス | 終端 | 備考 |
|---|---|---|---|
| N-BASIC (V1) | `0xF300` | `0xFAB7` (3000B) | 開始位置を任意に変更可 |
| N88-BASIC (V1/V2) | `0xF3C8` | `0xFF7F` (3000B) | $FF80 に揃う配置 |

V2 モードでは同じアドレス領域に「メイン RAM」「テキスト VRAM」
「グラフィック VRAM RGB 3 プレーン」の **5 バンク**が同居する。
Port `0x32` bit 4 が `0` のときテキスト VRAM、`1` のときメイン RAM。
GVRAM が選択されている (Port `0x5C-0x5E`) 状態では `0x32` bit 4 を
切り替えてもテキスト VRAM は出ない (グラフィックが優先)。

DMA 開始アドレスは V1 では `0x0000-0xFFFF` 任意、V1H/V2 では
`0xF000-0xFFFF` に制限される。`0x0000` を含む転送は内部処理の都合で
表示が乱れる (`0x0001` から開始は問題なし)。

### 1.2 1 行のメモリ配置

uPD3301 は **2 種類**のフォーマットを持つ。

**(a) トランスペアレントモード** (CRTC param 4 AT1-AT0=00 or 01)

```
1 行 = charsPerLine + attrsPerLine * 2  byte
        ┌──── 文字コード ────┐ ┌─ アトリビュートペア ─┐
        c0 c1 c2 ... c79     a0p a0v a1p a1v ... a19p a19v
                              └ 位置 ┘└ 値 ┘ × 20 個まで
```

- 80 桁時: 80 + 40 = **120 byte/行**
- アトリビュートペアは「適用開始 X 座標 (0-79)」「装飾値」の順
- 1 行最大 20 ペア。20 個未満でも 40 byte 領域は予約

**(b) ノントランスペアレントモード** (CRTC param 4 AT1-AT0=10)

```
1 行 = charsPerLine  byte (アトリビュートエリアなし)
        c0 c1 c2 ... c79
```

- 80 桁時: 80 byte/行
- **白黒固定** (アトリビュートで色指定不可、装飾も使えない)
- 「特殊制御文字 (SC) 有効」設定 (`AT1-AT0=10, SC=0`) では各行 +2 byte

> **ハマりどころ**: ノントランスペアレントは「char と attr が交互配置
> される」ものではなく「attribute が VRAM に存在しない」。
> Bubilator88 が当初 `bytesPerDMARow = charsPerLine * 2` (160) と
> 計算していたのは誤り (commit `fix/wiz5-nontransparent-text` で修正)。

### 1.3 桁・行数モード

PC-8801 のテキスト画面は 40x20, 40x25, 80x20, 80x25 の 4 種類。

**実装上の重要点** (vraminfo line 257-260):

> 基本は 80x25 で `(80 桁 + 40アトリビュート) * 25行 = 3000byte`
> のテキスト VRAM のまま。開始・終了アドレスがずれたりはしない。
>
> **40 桁の時は文字の書き込みは 1byte 飛ばしで行う。偶数アドレスのみ
> 有効で、奇数アドレスに書き込んでも表示されない。**
>
> 20 行の時は、25 行の内の 20 行のみ使用し、5 行分は単に表示されない。
>
> アトリビュートの位置指定についても、x 位置指定は偶数のみ有効で
> 奇数の場合は無視される。

つまり:
- **桁数の切替は描画フェーズの話**。VRAM 配置は常に 80 桁分の幅を
  確保する。40 桁モードでは奇数アドレスを「空白」と見なすのではなく
  「無視」して、偶数アドレスの内容を 16 ピクセル幅 (倍幅) で描画する。
- 20 行モードも同様で、配置は 25 行分のまま、表示が 20 行に制限される
  だけ。

桁数の切替元:
- **DIP SW1 bit 2** (起動時の初期値、Port `0x30` 入力 bit 2)
- **Port `0x30` 出力 bit 0** (動的切替: 1=80 桁, 0=40 桁)
- **CRTC RESET param 1 H6-H0** (`charsPerLine - 2`)

3 つを矛盾なく設定する必要がある。Port `0x30` のみ書き換えても
CRTC の charsPerLine が変わらないと表示が乱れる。

## 2. アトリビュート (装飾) の仕様

### 2.1 装飾値のフォーマット

カラーモード (`AT1-AT0=01,SC=0`) のとき bit 3 で色指定/装飾指定が
切り替わる。

```
カラー指定  (bit 3 = 1):                    装飾指定 (bit 3 = 0):
  bit 7 G                                     bit 7 0
  bit 6 R                                     bit 6 0
  bit 5 B                                     bit 5 アンダーライン
  bit 4 0=キャラクタ / 1=セミグラフィック      bit 4 アッパーライン
  bit 3 1                                     bit 3 0
  bit 2 0                                     bit 2 リバース
  bit 1 0                                     bit 1 ブリンク
  bit 0 0                                     bit 0 シークレット
```

白黒モード (`AT1-AT0=00 or 10`) では:

```
  bit 7 0=キャラクタ / 1=セミグラフィック
  bit 5 アンダーライン
  bit 4 アッパーライン
  bit 2-0 装飾効果 (リバース/ブリンク/シークレット)
```

### 2.2 装飾効果 (bit 2-0)

| bit210 | 効果 |
|---|---|
| 000 ($00) | ノーマル |
| 001 ($01) | シークレット |
| 010 ($02) | ブリンク |
| 011 ($03) | シークレット (= 001 と同じ) |
| 100 ($04) | リバース |
| 101 ($05) | リバースシークレット |
| 110 ($06) | リバースブリンク |
| 111 ($07) | リバースシークレット (= 101 と同じ) |

`bit 2 = リバース、bit 1 = ブリンク、bit 0 = シークレット`。

### 2.3 色 (bit 7-5, カラーモード)

| GRB | カラーアトリ値 ($08\|GRB<<5) |
|---|---|
| 000 黒 | `$08` |
| 001 青 | `$28` |
| 010 赤 | `$48` |
| 011 紫 | `$68` |
| 100 緑 | `$88` |
| 101 水色 | `$A8` |
| 110 黄色 | `$C8` |
| 111 白 | `$E8` |

### 2.4 アトリビュートの「掟」

vraminfo の有名な 5 つの制約。違反すると未定義動作:

1. **色 (bit 3=1) と装飾 (bit 3=0) は別々に指定**。同時設定不可、
   両方欲しければ 2 ペア消費。
2. **同じ開始位置で別アトリビュート適用は不可**。必ず X 座標をずらす。
3. **20 ペア制限**。1 行に 20 個まで。
4. **行頭ペアは X=0 強制**。`0xF418` (1 行目アトリビュートエリア
   先頭) に何を書いても X=0 と見なされる。
5. **若い X 順** (実装上は自動ソートされる)。

実機の挙動 (vraminfo 本人の追試):

> 開始位置は自動でソートされた上で順番に適用される。
> 属性値は最初から順番にピックアップされる。

つまり「位置リスト」と「属性値リスト」が分離して処理される。

### 2.5 起動時のアトリビュート初期値

| モード | 1 byte 目 | 2 byte 目 |
|---|---|---|
| 白黒 (起動直後) | `0x80` | `0x00` |
| カラー (`CONSOLE 0,25,0,1`) | `0x80` | `0xE8` |

> Bubilator88 では 0xE0 を default attr として使用しているが、これは
> ハードウェア初期値ではなく解釈時 fallback (BASIC ROM が初期化する
> パターンとは別)。

### 2.6 アンダー/アッパーラインの相互作用

vraminfo より:

- **ブリンク**: ライン部分は影響を受けない。文字本体だけ点滅。
- **リバース**: ライン部分が黒線になる (反転側で line bit が落ちる)。
- **シークレット**: ライン部分は影響を受けない。文字本体だけ黒塗り。

## 3. セミグラフィック

160x100 解像度の擬似グラフィック。テキスト VRAM の文字コード位置に
**ドットパターン** (1 byte = 8 ドット) を書き、対応するアトリビュートで
セミグラフィックビット (カラー時 bit 4、白黒時 bit 7) を立てる。

```
   bit 7 6 5 4 3 2 1 0
        ┌─┬─┐
        │0│1│  ← 上
        ├─┼─┤
        │2│3│
        ├─┼─┤
        │4│5│
        ├─┼─┤
        │6│7│  ← 下
        └─┴─┘
```

40 桁モードでは横方向に倍幅、奇数アドレスは無視。
アンダーライン/アッパーライン/ブリンクなどの装飾も適用可能。

## 4. CRTC (uPD3301) コマンド

コマンドは Port `0x51`、パラメータは Port `0x50` に書き込む。
コマンド・パラメータの書き込みは原則 V-Blank 期間中に行う。

### 4.1 RESET ($00)

リセットコマンド + 5 byte のパラメータ。

| Param | bit | 内容 |
|---|---|---|
| 1 | 7 (C/B) | 0=DMA バーストモード / 1=DMA キャラクタモード (実用は 1 のみ) |
| 1 | 6-0 (H6-H0) | 1 行あたりの文字数 - 2 (80 桁 → 78) |
| 2 | 7-6 (B1-B0) | カーソル/アトリビュート点滅周期 (4 段階, 後述) |
| 2 | 5-0 (L5-L0) | 1 画面あたりの行数 - 1 (25 行 → 24) |
| 3 | 7 (S) | 0=通常表示 / 1=一行おき表示 (skip line) |
| 3 | 6-5 (C1-C0) | カーソル形状 (後述) |
| 3 | 4-0 (R4-R0) | 文字あたりのライン数 - 1 |
| 4 | 7-5 (V2-V0) | 垂直帰線幅 - 1 (1〜8 の範囲) |
| 4 | 4-0 (Z4-Z0) | 水平帰線幅 - 2 (6〜33 の範囲) |
| 5 | 7-6 (AT1-AT0) | アトリビュートモード (後述) |
| 5 | 5 (SC) | 特殊制御文字有効/無効 |
| 5 | 4-0 (A4-A0) | 1 行あたり最大アトリビュート数 - 1 (1〜20) |

#### 4.1.1 カーソル形状 (C1-C0) と点滅周期 (B1-B0)

| C1-C0 | カーソル形状 |
|---|---|
| 00 | 点滅しないアンダーラインカーソル |
| 01 | 点滅するアンダーラインカーソル |
| 10 | 点滅しない■ブロックカーソル |
| 11 | 点滅する■ブロックカーソル |

| B1-B0 | カーソル / アトリビュート点滅周期 (frame) |
|---|---|
| 00 | 16 / 32 (速い) |
| 01 | 32 / 64 |
| 10 | 48 / 96 (デフォルト) |
| 11 | 64 / 128 (遅い) |

文字を構成するライン数が 13 以下の場合はアンダーラインが無効。

#### 4.1.2 アトリビュートモード (AT1-AT0, SC)

vraminfo (line 728-) より、許可される 5 通り:

| AT1-AT0,SC | 意味 |
|---|---|
| 00,0 | トランスペアレント白黒 + 特殊制御文字 (SC) 有効 |
| 00,1 | アトリビュート無し + SC 無効 |
| 01,0 | トランスペアレント・カラー |
| 10,0 | ノントランスペアレント白黒 + SC 有効 |
| 10,1 | ノントランスペアレント白黒 + SC 無効 |

**上記以外は禁止**。カラーは透過モードのみ、ノントランスペアレントは
白黒のみ。SC を有効にすると各行 +2 byte。

X88000M の enum 値はこの 3 ビットコードと一致:
```
ATTR_TRANSPARENT_MONO   = 0  // 00,0
ATTR_NONE               = 1  // 00,1
ATTR_TRANSPARENT_COLOR  = 2  // 01,0
ATTR_NONETRANSPARENT_SC = 4  // 10,0
ATTR_NONETRANSPARENT    = 5  // 10,1
```

#### 4.1.3 R4-R0 / V2-V0 の典型値

| 行数 | 周波数 | R4-R0 (文字ライン数 - 1) | V2-V0 (vretrace - 1) | Z4-Z0 (hretrace - 2) |
|---|---|---|---|---|
| 25 | 15kHz | 7 (= 8 - 1) | 6 (= 7 - 1) | 31 (= 32 - 2) |
| 25 | 24kHz | 15 (= 16 - 1) | 2 (= 3 - 1) | 25 (= 26 - 2) |
| 20 | 15kHz | 9 (= 10 - 1) | 5 (= 6 - 1) | 31 |
| 20 | 24kHz | 19 (= 20 - 1) | 1 (= 2 - 1) | 25 |

vraminfo より:
> VRTC = 水平周波数 ÷ (垂直周波数 × 1 文字のライン数) − 1 画面に表示する行数
> HRTC = 発振周波数 ÷ (水平周波数 × 8) − 1 行に表示する文字数

### 4.2 STOP DISPLAY ($00)

パラメータ無しの `OUT $51, 0`。テキスト画面+グラフィック画面とも
表示停止。表示再開には CRTC リセットからやり直し + 同期ビット
(Port `0x40` 出力 bit 3) の操作が必要。

### 4.3 START DISPLAY ($20 | DM)

`DM = 1` で全画面反転表示。アトリビュートのリバース指定が逆転する。

### 4.4 SET INTERRUPT MASK ($40 | ME<<1 | MN)

| bit | 内容 |
|---|---|
| 0 (MN) | 0=画面終了割込許可 / 1=禁止 |
| 1 (ME) | 0=特殊制御文字割込許可 / 1=禁止 |

DMAC オートロードモードでは両方とも 1 (禁止) にする。

### 4.5 READ LIGHT PEN ($60)

ライトペン座標を 2 byte 読み出し。Bubilator88 では未対応 (使用例稀)。

### 4.6 LOAD CURSOR POSITION ($80 | CM)

`CM = 1` でカーソル表示。X, Y を 2 byte (X: bit 6-0, Y: bit 5-0)。

### 4.7 RESET INTERRUPT ($A0)

DMA 要求を有効化。リセットコマンド + パラメータ設定の後に発行する。

### 4.8 RESET COUNTERS ($C0)

内部カウンタ全クリア。

### 4.9 READ STATUS (= Port `0x51` を read)

| bit | 内容 |
|---|---|
| 4 (VE) | 1=画面表示有効 |
| 3 (U) | 1=DMA アンダーラン (1 行転送未完了)。発生で画面停止 |
| 2 (N) | 1=特殊制御文字割込発生 |
| 1 (E) | 1=表示終了割込発生 |
| 0 (LP) | 1=ライトペン入力あり |

通常のオートロードモードでは N/E は使われない (常に 0)。

## 5. DMAC (uPD8257)

### 5.1 ポート

| Port | チャネル | 内容 |
|---|---|---|
| `0x60`-`0x61` | ch.0 | 5"DMA ディスク |
| `0x62`-`0x63` | ch.1 | 8"DMA ディスク |
| `0x64`-`0x65` | ch.2 | **テキスト** |
| `0x66`-`0x67` | ch.3 | テキストオートロードバッファ |
| `0x68` | 制御 | コントロール / ステータス |

各チャネル 2 ポート構成:
- 偶数 (`0x60` 等): 開始アドレス Lo/Hi (フリップフロップで切替)
- 奇数 (`0x61` 等): TC Lo / [動作モード bit 7-6 + TC Hi bit 5-0]

動作モード: `00`=ベリファイ / `01`=ライト / `10`=リード。

### 5.2 Port 0x68 (コントロール / ステータス)

**入力 (ステータス)**:

| bit | 内容 |
|---|---|
| 0-3 | TC ステータス ch.0-3 (read で 0 クリア) |
| 4 | UPDATE フラグ (オートロード ch.3 → ch.2 コピー時に 1) |

**出力 (制御)**:

| bit | 内容 |
|---|---|
| 0-3 | ch.0-3 イネーブル |
| 4 | 回転優先 |
| 5 | 拡張ライト |
| 6 | TC ストップ |
| 7 | オートロード (ch.3 → ch.2) |

V2 モードでテキスト DMA を使う場合、オートロードモード (bit 3 + 7
セット) で ch.3 にパラメータ、ch.2 が自動再ロードされる構成。

### 5.3 1 行あたりの転送量

ノントランスペアレントモードを含めて、CRTC 設定との対応:

```
bytesPerDMARow = charsPerLine + (transparent ? attrsPerLine * 2 : 0)
```

- トランスペアレント: 80 + 40 = **120 byte/行**
- ノントランスペアレント: **80 byte/行** (attribute は VRAM になし)
- 特殊制御文字有効: 上記 +2 byte (vraminfo line 738)

BubiC `pc88.cpp:2836` でも同じ計算:
```cpp
dmac.run(2, 80 + crtc.attrib.num * 2);  // attrib.num = 0 in non-transparent
```

### 5.4 V-Blank 待ち (典型コード)

vraminfo より:

```z80
WaitVBlank:
    in   a,($40)
    and  $20
    jr   nz,WaitVBlank      ; bit 5: ディスプレイサイクルになるまで待つ
.wait:
    in   a,($40)
    and  $20                ; ディスプレイ → V-Blank 立ち上がり待ち
    jr   z,.wait
    ret
```

V-Blank 割込 (`0x40` bit 5 立ち上がり、IM2 vector レベル 1) と併用すると
取りこぼしの恐れあり。どちらか片方に統一すべき。

## 6. 関連 I/O ポート

### 6.1 Port `0x30` (システム制御)

**入力** (DIP SW1):

| bit | 内容 |
|---|---|
| 0 | 起動時 BASIC モード (0=N / 1=N88) |
| 1 | 起動時モード (0=ターミナル / 1=BASIC) |
| 2 | 起動時テキスト桁数 (0=80 / 1=40) |
| 3 | 起動時テキスト行数 (0=25 / 1=20) |
| 4 | S パラメータ (0=有効 / 1=無効) |
| 5 | DEL コード処理 (0=処理 / 1=無視) |

**出力**:

| bit | 内容 |
|---|---|
| 0 | CRT 桁数 (0=40 / 1=80) |
| 1 | CRT モード (0=カラー / 1=白黒) |
| 2 | CMT キャリア (0=Mark 2400Hz / 1=Space 1200Hz) |
| 3 | CMT モーター (0=Off / 1=On) |
| 5-4 | USART チャネル (00=CMT600 / 01=CMT1200 / 10=RS-232C 非同期 / 11=同期) |

### 6.2 Port `0x31` (BASIC モード / グラフィック)

**入力** (DIP SW2):

| bit | 内容 |
|---|---|
| 7 | BASIC モード (0=V2 / 1=V1) |
| 6 | スピード (0=V1S 標準 / 1=V1H 高速) |
| 5 | 通信方式 (0=半二重 / 1=全二重) |
| 4-0 | RS-232C 設定 |

**出力**:

| bit | 内容 |
|---|---|
| 0 | グラフィックモード (0=640×400×1 / 1=640×200×3) |
| 1 | RAM モード (0=ROM/RAM / 1=64K RAM) |
| 2 | ROM モード (0=N88-BASIC / 1=N-BASIC) |
| 3 | グラフィックディスプレイ (0=非表示 / 1=表示) |
| 4 | グラフィック白黒/カラー (0=白黒 / 1=カラー) |
| 5 | 専用高解像度 CRT 行数 (0=20 行 / 1=25 行) |

CRTC の行数設定とこの Port の bit 5 が独立しており、**両者を独立に
設定すると不思議な表示**になる:

- CRTC=25 + Port0x31=20: 文字縦幅は 20 行サイズ (大), 表示は 25 行分
- CRTC=20 + Port0x31=25: 文字縦幅は 25 行サイズ (小), 表示は 20 行分

### 6.3 Port `0x32` (拡張制御)

| bit | 内容 |
|---|---|
| 7 | サウンド割込マスク (0=許可 / 1=禁止) |
| 6 | GVRAM アクセス (0=独立 / 1=ALU) |
| 5 | カラーモード (0=デジタル / 1=アナログ) |
| 4 | 高速 RAM/メイン (0=テキスト VRAM / 1=メイン RAM) |
| 3-2 | 画面出力モード (10=アナログ RGB が標準) |
| 1-0 | 内部 EROM 選択 (00-11 でバンク 0-3) |

### 6.4 Port `0x40` (制御信号)

**入力**:

| bit | 内容 |
|---|---|
| 5 | VRTC (0=ディスプレイ / 1=垂直帰線) |
| 4 | カレンダ CDO |
| 3 | DISK ブートモード (実機/BubiC/QUASI88 では常に 0) |
| 2 | CMT データキャリア検出 (DCD) |
| 1 | 高解像度 CRT (0=24kHz / 1=15kHz) |
| 0 | プリンタ BUSY |

> bit 3 については Bubilator88 では BubiC/QUASI88 準拠で常に 0
> (ROM/DISK strap DIP は port 0x40 に出さない)。

**出力**:

| bit | 内容 |
|---|---|
| 7 | ポート波形出力 (mk2 以降) |
| 6 | 汎用出力 |
| 5 | BEEP (2400Hz) |
| 4 | グラフィックハイスピードモード (V1S のみ) |
| 3 | CRT I/F 同期ビット |
| 2 | カレンダクロックシフト |
| 1 | カレンダクロックストローブ |
| 0 | プリンタストローブ |

### 6.5 Port `0x52` (背景色 / ボーダー)

白黒モードのみ有効。

| bit | 内容 |
|---|---|
| 6 | 背景色 G |
| 5 | 背景色 R |
| 4 | 背景色 B |
| 2 | ボーダー G (mk2 以降は無効) |
| 1 | ボーダー R (mk2 以降は無効) |
| 0 | ボーダー B (mk2 以降は無効) |

### 6.6 Port `0x53` (プレーン / テキスト非表示)

| bit | 内容 |
|---|---|
| 3 | 白黒時 G プレーン非表示 |
| 2 | 白黒時 R プレーン非表示 |
| 1 | 白黒時 B プレーン非表示 |
| 0 | テキスト非表示 (カラー時も有効) |

bit 3-1 はカラーモード時は無視 (BubiC 確認済み)。

### 6.7 Port `0x54-$5B` (パレット)

`$32` bit 5 でデジタル/アナログ切替、レジスタは共通。

**デジタル**:

| bit | 内容 |
|---|---|
| 2 | G プレーン on/off |
| 1 | R プレーン on/off |
| 0 | B プレーン on/off |

**アナログ**:

| bit | 内容 |
|---|---|
| 7-6 | `00`=R, B 設定 / `01`=G 設定 |
| 5-3 | R bit 2-0 |
| 2-0 | B (G) bit 2-0 |

`$54` bit 7-6 = `10`/`11` で BG パレット (アナログ背景色)。

## 7. グラフィック VRAM

### 7.1 アドレスとプレーン

```
0xC000 ┌──────────────┐
       │ B / R / G プレーン (200 line)│  3 プレーン × 16,000 byte
       │ または                       │  bank 切替 ($5C/$5D/$5E)
       │ 上半分 B + 下半分 R (400 line) │
0xFE7F ├──────────────┤
       │ 表示外領域                   │  ALU は有効
0xFFFF └──────────────┘
```

- 1 プレーン = 80 byte × 200 line = 16,000 byte
- 1 byte = 8 ドット (横並び)
- 200/400 ライン切替: Port `0x31` bit 0 + bit 4 (=0 白黒) で 400
- カラー: 200 ライン専用 (Port `0x31` bit 4 = 1)

### 7.2 アクセスモード

**独立アクセス** (Port `0x32` bit 6 = 0):

Port `0x5C-0x5E` でプレーン bank 切替、`0x5F` でメイン RAM 復帰。
書き込む値は何でも良い。`0x5C` の **入力**で現在の選択状態が読める
(bit 2-0 = G/R/B 選択フラグ)。

V1 モードでは VRAM bank を選択しているだけで CPU 実行速度が低下する
(メモリ wait state、`Pc88Bus.swift:215` v1sMemWait)。

**ALU アクセス** (Port `0x32` bit 6 = 1):

Port `0x35` で挙動を制御:

| bit 7 | 内容 |
|---|---|
| 0 | (ALU on のまま) メイン RAM / テキスト VRAM アクセス |
| 1 | GVRAM アクセス |

| bit 5-4 | 内容 |
|---|---|
| 00 | Port `0x34` の論理演算設定でライト |
| 01 | 3 プレーン同時 R/W (8bit バッファ経由、16bit 命令不可) |
| 10 | R プレーン → B プレーンへ転送 |
| 11 | B プレーン → R プレーンへ転送 |

| bit 2-0 | 内容 |
|---|---|
| GRB | 比較データ (3 プレーン同時 R で「この色のドットだけ 1」マスクが返る) |

Port `0x34` (ALU 論理演算指定):

| bit 7-6, 5-4, 3-0 | プレーン G/R/B にそれぞれ 2bit 指定 |
|---|---|

2bit 値:
| 値 | 演算 |
|---|---|
| 00 | NOT AND (= ビットリセット / マスク) |
| 01 | OR (ビットセット) |
| 10 | XOR (ビット反転) |
| 11 | NOP (何もしない) |

「上書き」は仕様上存在しないので、`NOT AND` で消去 → `OR` で書き込みが
キャラクタ重ね合わせの定石。

ALU モードに入ると独立アクセスのプレーン選択状態はリセットされ、
ALU を解除しても復元されない。

## 8. 表示中の動的切替

vraminfo によると以下は実機で**有効**:

- **200 ↔ 400 ライン切替** (画面の上下で混在可能)
- **グラフィック画面の表示 ON/OFF**
- **DMA 開始アドレス・転送量変更**による 27 色表示
  (2 画面分の VRAM を交互に表示する MAGICAL COLOR テクニック)
- **行数を 32/50 に増やす**ことによる 2x2/2x3 ドットセミグラ着色
- **パレットを hblank 毎に書き換える**ことで 8 色超え

実機で**無効**:

- Port `0x32` デジタル ↔ アナログ切替 (これはレジスタ書き込みモード
  を変えるだけ)
- Port `0x30` 80/40 桁切替や白黒/カラー切替 (画面が乱れる)

## 9. アトリビュートとグラフィックの相互作用

vraminfo §200 行ライン白黒モード関連:

- 白黒 GVRAM の「白」部分は **テキストアトリビュート色**を反映する。
  これが PC-8801 で 400 ライン白黒モードでも**色付き漢字**が出せる
  仕掛け。
- デジタルパレットを変更しても影響なし (常にデジタル 8 色)。
- アナログパレットなら設定値が反映される。

```
カラー G + 白黒 200 ライン 組み合わせ表 (vraminfo より):
  T=0 (白黒)    T=1 (カラー)
  G=0 (col 200) ┐
  G=1 (bw 200)  ├ 6 通り存在し、組み合わせで描画結果が異なる
  G=2 (bw 400) ─┘
```

## 10. 特殊制御文字

ノントランスペアレントモードまたは `AT1-AT0=00, SC=0` のとき、
アトリビュートエリアに `[0xE0 or 0x60][制御 byte]` の形式で
**割込/表示停止/DMA 停止**を引き起こせる。

制御 byte の bit:

| bit | 内容 |
|---|---|
| 2 (I) | 検出時に CRTC ステータス N=1, 割込発生 |
| 1 (V) | 検出時から最終行まで表示停止 |
| 0 (D) | 検出時から最終行まで DMA 停止 |

オートロードモードでは I は意味を持たない (常に N=0)。
使用例は稀なので Bubilator88 では未実装 (`#41`, audit memory)。
