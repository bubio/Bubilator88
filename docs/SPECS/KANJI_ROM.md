# PC-8801 漢字ROM

漢字ROM はメモリマップではなく I/O ポート経由でアクセスする。
第一水準 (JIS 第一水準漢字, 128KB) と第二水準 (JIS 第二水準漢字, 128KB) がある。

---

## I/Oポート

### 第一水準

| Port | R/W | 説明 |
|------|-----|------|
| 0xE8 | W | 漢字ROMアドレス 下位8ビット設定 |
| 0xE8 | R | フォントの右側8ビット (偶数ラインは右半分) |
| 0xE9 | W | 漢字ROMアドレス 上位8ビット設定 |
| 0xE9 | R | フォントの左側8ビット (奇数ラインは左半分) |
| 0xEA | W | 読み出し開始 |
| 0xEB | W | 読み出し終了 |

### 第二水準

| Port | R/W | 説明 |
|------|-----|------|
| 0xEC | W | 漢字ROMアドレス 下位8ビット設定 |
| 0xEC | R | フォントの右側8ビット |
| 0xED | W | 漢字ROMアドレス 上位8ビット設定 |
| 0xED | R | フォントの左側8ビット |

---

## アクセス手順

1. Port 0xE8 に漢字ROMアドレスの下位8ビットを設定
2. Port 0xE9 に漢字ROMアドレスの上位8ビットを設定
3. Port 0xEA に何かデータを出力 → 読み出し開始
4. 8クロックのウェイト
5. Port 0xE9 から文字フォントデータ (左側) を入力
6. Port 0xE8 から文字フォントデータ (右側) を入力
7. Port 0xEB に何かデータを出力 → 読み出し終了
8. 漢字ROMアドレスを +1 してループ

※ FH 以降では手順 3, 4, 7 は不要。  
※ 初回アクセス前に Port 0xEB に何かを出力しておく。

## JISコード → 漢字ROMアドレス変換

漢字フォントは 16×16 ピクセル。各文字は 16行 × 2バイト = 32バイト。

### 半角文字 (JIS 0x0020-0x00FF)

- 8×16 ピクセル (半角幅)
- 漢字ROMアドレス: JIS の下位ビットを並べ替え、下位に `000` を付加

```
JIS:      0 0 0 0 0 0 0 0 | b7 b6 b5 b4 b3 b2 b1 b0
ROM addr: 0 0 0 0 0 b7 b6 b5 b4 b3 b2 b1 b0 0 0 0
```

### 1/4角文字 (JIS 0x0100-0x01FF)

- 4×8 ピクセル

```
JIS:      0 0 0 0 0 0 0 1 | b7 b6 b5 b4 b3 b2 b1 b0
ROM addr: 0 0 0 0 0 1 1 0 | b7 b6 b5 b4 b3 b2 b1 b0 0 0
```

### 非漢字 (JIS 0x2120-0x277F)

- JIS 第一水準の記号類

```
JIS:      0 0 1 r4 r3 0 | c7 c6 c5 c4 c3 c2 c1 c0
ROM addr: 0 0 r4 r3 c7 c6 c5 c4 c3 c2 c1 c0 0 0 0 0
```

### 第一水準漢字 (JIS 0x3020-0x4F5F)

```
JIS:      0 r6 r5 r4 r3 r2 r1 0 | c7 c6 c5 c4 c3 c2 c1 c0
ROM addr: r6 r5 r4 r3 r2 r1 c7 c6 c5 c4 c3 c2 c1 c0 0 0 0 0
```

### 第二水準漢字 (JIS 0x5020-0x7F7F)

第一水準と同様のマッピングだが、第二水準 ROM (Port 0xEC-0xED) でアクセスする。

---

## フォントデータ構造

各文字のフォントは上から順に16行。各行は左側8ビット + 右側8ビット = 16ビット幅。

```
行0:  [Port 0xE9 = 左8bit] [Port 0xE8 = 右8bit]
行1:  [Port 0xE9 = 左8bit] [Port 0xE8 = 右8bit]
  :
行15: [Port 0xE9 = 左8bit] [Port 0xE8 = 右8bit]
```

半角・1/4角文字はフォント幅が狭いが、ROM上は同じ16ビット幅で格納される。偶数ライン/奇数ラインで左右の読み出しポートが分かれる。
