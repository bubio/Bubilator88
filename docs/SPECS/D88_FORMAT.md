# D88 ディスクイメージフォーマット

P88SR.EXE (PC-8801mkII SR emulator) 由来のディスクイメージフォーマット。
拡張子は `.D88` または `.d88`。1ファイルに複数のディスクイメージを連結可能。

---

## 全体構成

```
+---------------------+
|      ヘッダ          |  0x2B0 バイト固定
+---------------------+
| トラック (Track 0)   |  可変長
+---------------------+
| トラック (Track 1)   |  可変長
+---------------------+
|         :            |
+---------------------+
```

---

## ヘッダ (0x2B0 = 688 バイト)

| Offset | Size | 内容 |
|--------|------|------|
| 0x00 | 17 | ディスク名 (ASCII + '\0') |
| 0x11 | 9 | 予約 (0x00) |
| 0x1A | 1 | ライトプロテクト (0x00=なし, 0x10=あり) |
| 0x1B | 1 | ディスク種別 (0x00=2D, 0x10=2DD, 0x20=2HD) |
| 0x1C | 4 | ディスクサイズ (ヘッダ + 全トラック, LE) |
| 0x20 | 4×164 | トラックオフセットテーブル (Track 0-163, LE) |

### トラックオフセットテーブル

各エントリはヘッダ先頭からの絶対バイトオフセット (4バイト LE)。
存在しないトラックは 0x00000000。

- Track 0 のオフセットは通常 0x000002B0
- 2D の場合、Track 84-163 は存在しないため 0

---

## トラック

トラックは連続するセクタの並び。セクタ数はトラックごとに異なりうる。

---

## セクタ (16バイトヘッダ + データ)

| Offset | Size | 内容 |
|--------|------|------|
| 0x00 | 1 | ID: C (シリンダ番号) |
| 0x01 | 1 | ID: H (ヘッド番号) |
| 0x02 | 1 | ID: R (セクタ番号) |
| 0x03 | 1 | ID: N (セクタサイズコード) |
| 0x04 | 2 | このトラック内のセクタ数 (LE) |
| 0x06 | 1 | 記録密度 (0x00=倍密度, 0x40=単密度) |
| 0x07 | 1 | DELETED MARK (0x00=通常, 0x10=DELETED) |
| 0x08 | 1 | ステータス |
| 0x09 | 5 | 予約 |
| 0x0E | 2 | データサイズ (バイト数, LE) |
| 0x10 | 可変 | セクタデータ (0x0E のバイト数分) |

### セクタサイズコード (N)

| N | バイト数 |
|---|---------|
| 0 | 128 |
| 1 | 256 |
| 2 | 512 |
| 3 | 1024 |

### ステータス

PC-98x1 BIOS 互換のステータスコード:

| 値 | 意味 |
|-----|------|
| 0x00 | 正常 |
| 0x10 | 正常 (DELETED DATA) |
| 0xA0 | ID CRC エラー |
| 0xB0 | データ CRC エラー |
| 0xE0 | アドレスマークなし |
| 0xF0 | データマークなし |

---

## 標準フォーマット

| パラメータ | 2D | 2DD | 2HD |
|-----------|-----|-----|-----|
| トラック数 | 40 | 80 | 77 |
| サイド数 | 2 | 2 | 2 |
| セクタ/トラック | 16 | 16 | 26 |
| セクタサイズ | 256B (N=1) | 256B (N=1) | 256B (N=1) |
| 総トラック数 | 80 (0-79) | 160 (0-159) | 154 (0-153) |

### トラック番号の計算

D88 のトラックオフセットテーブルのインデックス:
- `trackIndex = cylinder * 2 + head`

---

## 複数ディスク連結

1つのファイルに複数のディスクイメージを連結可能。
各ディスクの `ディスクサイズ` フィールドで次のディスクの先頭位置が分かる。

---

## バイトオーダー

ヘッダ内の数値フィールド (ディスクサイズ, トラックオフセット, セクタ数, データサイズ) は全てリトルエンディアン。
