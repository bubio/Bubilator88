# Cassette Tape — 手動検証チェックリスト

カセットテープ (CMT) 周りの実装を変更した際の目視確認手順。自動化は
CLOAD → RUN の入力タイミングがタイトルごとにばらつくため断念し、
目視 + ユニットテストの 2 段構成で運用する。

- **ユニットレベル**: `Packages/EmulatorCore/Tests/EmulatorCoreTests/`
  の `I8251Tests` / `CassetteDeckTests` / `MachineTests` (save state
  roundtrip) — 計 20+ 件。`swift test` 前に全通過を確認。
- **手動レベル**: 下記タイトルをアプリで目視確認。

## テストイメージの所在

`/Volumes/CrucialX6/roms/PC8801 ROM 1240本/ＰＣ８８/tape/<タイトル>/<タイトル>.{cmt,t88}`
(計 32 タイトル、`.cmt` 24 / `.t88` 10)。

## スモーク手順

1. **アプリ起動**。Boot Mode は **V1H** (確実に N88-BASIC V1 へ入る)。
2. N88-BASIC Ok プロンプトが出るのを確認。
3. `File > Tape > Open Tape...` (⌘⇧T) で `.cmt` または `.t88` を選択。
4. ステータスバーに波形アイコンと読み込み中のテープ名が表示されることを確認。
5. BASIC に `CLOAD` + Return を入力。
6. 読み込み中モータ音は出ない (実装範囲外)。数秒〜数十秒後に `Ok.` 復帰。
7. `RUN` + Return でタイトル画面が出ることを確認。

## 目視確認対象 (最低限)

エッジケースを押さえる狙い:

| タイトル | 形式 | 確認ポイント |
|---------|------|-------------|
| ドアドア | CMT | 単一 BASIC ヘッダの標準ケース |
| アルフォス | T88 | T88 data carrier タグの処理 |
| パックマン | CMT | マシン語 (0x9C ヘッダ) パス |
| メネシス88 | T88 | T88 複数データブロック |
| タイムシークレット | T88 | BASIC + マシン語混在 |

上記 5 タイトルがタイトル画面に到達すれば、基本パスはカバーされる。
該当タイトルが動かなくなった場合は、直前の変更を疑う。

## SaveState 手動検証 (追加推奨)

1. 上記のいずれかをロード → タイトル画面まで進める。
2. `⌘S` でクイックセーブ。
3. アプリ再起動 → `⌘L` でクイックロード。
4. タイトル画面状態が復元されることを確認。セーブステートに `CMT `
   セクションが含まれているため、テープバッファと再生位置が復元される。

## 期待動作 (既知の制限)

- **CSAVE は未実装**。BASIC 側が `Syntax error` にはならないが書き込みは
  されない。録音は対応予定なし。
- **FSK 波形は模擬せず**、バイト単位で I8251 に注入。タイミング依存の
  プロテクトタイトルは想定外。
- **速度は実機 ×1.7 程度** (5000 T-state/バイト)。完全な瞬間ロードでは
  ないのは N88-BASIC ROM の同期検出ループに追従するため (`docs/SPECS/
  CASSETTE_TAPE.md` 参照)。

## 壊れた場合の手がかり

- ステータスバーにテープアイコンが出ない → `EmulatorViewModel+Tape.swift`
  の `mountTape(url:)` を確認
- `CLOAD` で `Device I/O Error` → `Pc88Bus.swift` の Port 0x30 bit 3/5
  ルーティングか `CassetteDeck.tick` の `bytePeriodTStates` を確認
- 途中で止まる → `CassetteDeck.dcd` の遷移条件 (現状はバッファ残量のみで
  判定する簡易モデル; BubiC はキャリア区間の詳細遷移を模擬)
